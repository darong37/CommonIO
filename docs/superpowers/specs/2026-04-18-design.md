## グループ A：`at()` 正式 API 化

### API

`at()` → `callers`

### 動作

- `caller` ベースで呼び出し履歴を収集し、配列リファレンス（`callers`）を返す。
- 最上位の `.pl` ファイルをレベル 0 とし、現在の呼び出し位置まで昇順に並べる。
- `.pl` が見つからない場合は取得できた最上位フレームをレベル 0 とする。
- CommonIO 自身の内部フレームは含めない。

### 各要素（`caller_info`）

```perl
{
    file       => "script.pl",           # ベースネーム
    path       => "/abs/path/script.pl", # フルパス
    line       => 42,
    subroutine => "main::foo",           # Package::name 形式
}
```

### 用途

グループ B のログファイル名自動決定でも使う共通 API。

## グループ B：ログファイル自動決定 + `setLogFile` 廃止

### 環境変数 `LOGDIR`

- 必須。未設定または空のとき `use CommonIO` のロード時点で `die` する。
- ログファイルはすべて `LOGDIR` 配下に出力する。

### ログファイル名の自動決定

- `at()` を使って最上位の `.pl` のベース名（拡張子 `.pl` を除いたもの）を取得する。
- `.pl` が見つからない場合は `at()` の最上位フレームのベース名を使う。
- ファイル名形式：`<script_basename><MMDDHHMM>.log`（ローカル時刻・月日時分を各 2 桁）
  - 例：`myscript04181530.log`

### タイミング

- モジュールロード時（`use CommonIO`）に `LOGDIR` チェックとファイル名決定を行う。
- `log` が最初に呼ばれる前からファイルへ追記できる状態を保証する。

### `setLogFile` の廃止

- 公開 API からは削除する。
- 内部専用として残すかどうかは実装時に判断する。

## グループ C：`out_file` API 追加

### API

`out_file($path, $text)` または `out_file($path, $lines)` → なし

### 動作

- 同じ `path` への 1 回目は `write_file` 相当（上書き）。
- 同じ `path` への 2 回目以降は `append_file` 相当（追記）。
- `path` の指定形式・`encoding` / `eol` の扱いは `write_file` / `append_file` と同じ。

### 既出管理

- ファイルパス文字列をキーとするハッシュで管理する。
- 値はそのパスへの書き込み回数。
- 起動時の親プロセス PID と組で保持する。
- 同じ PID の `END` ブロックが動いたときだけハッシュをクリアする。
- `fork` 後の子プロセス終了ではクリアしない。

### `END` ブロック

将来のレポート機能のため、以下の英語コメントを `END` ブロック内に残す：

```perl
END {
    if ($$ == $initial_pid) {
        # Future: report written files and write counts here.
    }
}
```
