## 1. `_print_console_line` の廃止

### 問題

現在の `_print_console_line` は内部で `binmode STDERR, ':raw'` を実行している。
これにより `setup_console` が設定した STDERR のエンコード層が毎回 `log` 呼び出しのたびに破壊される。

### 修正方針

- `_print_console_line` サブルーチンを**削除**する。
- `log` 内の `_print_console_line($line)` 呼び出しを `print STDERR $line;` に置き換える。
- エンコード変換の責務は `setup_console` が設定した Perl のエンコード層に委ねる。

### 前提

- `setup_console` を呼ばない状態でも `log` は動作するが、Wide character 警告が出る可能性がある。これはユーザー責任とする。
- `I18N::Langinfo` の `use` は `_console_encoding_name`（`setup_console` 内）でまだ使用するため削除しない。

## 2. `dp` API の追加

### インターフェース

```
dp(@args)
```

- 戻り値：なし
- 出力先：STDERR（Data::Printer のデフォルト出力先）
- `@EXPORT_OK` に追加する

### 引数処理ルール

| 呼び出し | 内部処理 | 備考 |
|---|---|---|
| `dp()` | 何もしない | 引数なしは無視 |
| `dp($scalar)` | `p($scalar)` | スカラはそのまま渡す |
| `dp(\@array)` | `p(\@array)` | リファレンス1つはそのまま渡す |
| `dp(\%hash)` | `p(\%hash)` | リファレンス1つはそのまま渡す |
| `dp(@array)` | `p([@array])` | 複数引数は配列リファレンスにまとめる |
| `dp(%hash)` | `p([@_])` | 複数引数（偶数）も配列リファレンスにまとめる |

判定ロジック：
- 引数が0個 → 何もしない
- 引数が1個かつリファレンス → そのまま `p` に渡す
- それ以外（複数引数、または1個のスカラ） → `[@_]` で配列リファレンス化して渡す

### Unicode 表示設定

Data::Printer を `unicode_flag => 1` で呼び出し、内部文字列をエスケープせず生の Unicode（漢字等）で表示する。

### 出力エンコーディング

Data::Printer がデフォルトで STDERR へ書く。`setup_console` 設定済みの STDERR エンコード層が変換を担う。`dp` 側では `binmode` や `encode` を行わない。

### 依存関係

- `use Data::Printer qw(p)` を追加する。

### `dp` テスト

| テストケース | 確認内容 |
|---|---|
| `dp()` | 例外が発生しない |
| `dp($scalar)` | 例外が発生しない |
| `dp(\@array)` | 例外が発生しない |
| `dp(@array)` 複数引数 | 例外が発生しない |
